User Experience

To demonstrate an account holder's experience interacting with SimpleFIN, we have created a fake banking portal (for a ficticious bank named FakeBank) and a fake financial management tool named FakeAccountant.

Typically, an account holder will use a SimpleFIN Reader (FakeAccountant in this case), which will direct the user to the SimpleFIN Server (FakeBank in this case) to acquire a token.

To begin the demo, visit FakeAccountant.

Please note that there is no backend server or database powering FakeBank or FakeAccountant; all information is stored in your browser and the SimpleFIN protocol is simulated. The goal of this example is to indicate a user's experience, not how to implement the SimpleFIN protocol.
