SimpleFIN

Getting transaction information from your bank should be easy and secure. SimpleFIN (or just SFIN) stands for Simple Financial Interchange and aims to make read-only financial interchange simple.

The SFIN standard defines:

  1. a data format for financial transaction information
  2. the interface HTTP servers should implement to serve SFIN data, and
  3. the authentication mechanism used in communicating with the HTTP servers.

Data Format

Data is encoded as JSON strings. Following are the objects specified by this standard:

Organization

Name Type Example Required Description
domain string "example.com" required The domain name of the organization.
sfin-url string "https://www.example.com/sfin/" optional URL of organization's SFIN root endpoint. If provided, the domain of the URL must match domain.

Account

Name Type Example Required Description
org Organization {% highlight javascript %}{"domain": "example.com", "sfin-url": "https://www.example.com/sfin/"}{% endhighlight %} required Organization of this account. If this is a bank account, this field will be the bank's organization.
id string "123" required An ID that uniquely identifies an account within the account's organization. This does not have to be the same value that a person would recognize as their account number.
name string "Money-Market Checking" required A name that uniquely describes an account. This name should be chosen so that a person can easily associate it with only one of their accounts within an organization.
currency string "GBP" required Currency of account. XXX find list of acceptable values.
balance numeric string "22.33" required The balance of the account as of last-transaction-id.
available-balance numeric string "15.33" optional The available balance of the account as of last-transaction-id. This may be omitted if it is the same as balance.
last-transaction-id string "83940394098343ABCFKDSDF89" required ID of the most recently recorded Transaction on this account.

Transaction

Name Type Example Required Description
id string "9990203-3840394" required An ID that uniquely describes a transaction within an Account. An organization may reuse transaction ids for different accounts, but may never reuse a transaction id within an account.
posted ISO GMT date string "2015-01-01T10:00:03.23434" required Timestamp transaction happened. The timezone must be GMT.
amount numeric string "90.28" required Amount of transaction. Positive numbers indicate money being deposited into the account.
memo string "Bob's Grocery Store - 22 S. Main" required A human-readable description of the source (in the case of a deposit) or destination (in the case of a withdrawl) of the money.
sic string "2839" optional XXX SIC of payee

HTTP Servers

HTTP servers must require SSL for all SFIN communication. Servers must provide the following set of URL paths:

Method Path What it's for
GET /accounts Fetch accounts. Among other things, a successful response will contain a list of accounts available to the requester.
GET /accounts/:id Fetch transactions. Successful responses will contain account information and a list of transactions.

Fetch accounts

 
GET /accounts

Fetch a list of all accounts available to the requester. Here is an example response body:

{% highlight javascript %}{ "accounts": [ { "org": { "domain": "mybank.com", "sfin-url": "https://sfin.mybank.com/" }, "id": "2930002", "name": "Savings", "currency": "USD", "balance": "100.23", "available-balance": "75.23", "last-transaction-id": "AO334" } ] ]{% endhighlight %}

Fetch transactions

 
GET /accounts/:id

Fetch a list of the most recent transactions for a particular account (specified by the :id). If the response from GET /accounts indicates that there's an account with id of 1392, the transactions for that account will be found at /accounts/1392.

Servers must respond with all transactions for at least the past 60 days, though more than 60 days of transactions may be returned.

Transactions must be sorted by id, with the latest transaction last.

Here is an example response body:

{% highlight javascript %}{ "transactions": [ { "id": "AO334", "posted": "2001-01-01T09:22:33.2343", "amount": "-22.33", "memo": "John's Pizza Pie Castle, Cafe and Spa", "sic": "2354" }, { "id": "AO335", "posted": "2001-01-05T10:33:29.82", "amount": "-5.02", "memo": "Frodo's Ring Repair", "sic": "9003" } ] }{% endhighlight %} Optional query parameters
Name Example Description
since-transaction ?since-transaction=AO335 If given, only transactions with an id newer than and not including since-transaction will be returned.
